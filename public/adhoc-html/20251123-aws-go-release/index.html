なぜ今、インフラエンジニアがGoなのか？

「言われた通り作るだけ」からの脱却。アプリの知識がインフラの質を変える。

サーバー構築の質が変わる

アプリがどう動くかを知らないと、最適なインフラ構成は作れない。「言われたスペックで作る」から、アーキテクチャを提案できるエンジニアへ。

DevOps & CI/CD

テストやビルドの「辛さ」を理解することで、開発者に真に寄り添ったパイプライン設計が可能に。自動化の価値を最大化する。

Serverless & Lambda

Lambdaはロジックそのもの。クラウドエンジニアもコードが書けないと手が出せない領域が増えている。Goなら高速起動でサーバーレスに最適。

Special Talk

AI時代に問う「基礎力」の価値

なぜ今、AWS学習にGo言語が必要なのか。制作者ジェバンニ氏に迫る。

Topic 1: インフラとアプリの境界線
くろかわ

「インフラエンジニアにとって『なぜ今Goなのか？』。最近はDevOpsやサーバーレスの現場で、アプリの知識がないと『言われた通り作るだけ』になりがちですよね。」

ジェバンニ

「まさにその通りです。サーバー構築、DevOps、サーバーレス。どの業務でも、アプリの中身を知らないと良い仕事ができない時代です。

特にCI/CDパイプラインを構築する際、アプリ開発者の『テストやビルドの辛さ』を理解していないと、現場で使われないパイプラインを作ってしまう。Go言語を通してその感覚を掴んでほしいんです。」

Topic 2: AI時代の「責任」と「基礎」
ジェバンニ

「AIがコードを書く時代ですが、AIはエラー処理やログのセキュリティ配慮（個人情報の出力など）を平気で間違えます。
その正誤を見抜く『目』を持つには、やはり基礎が必要です。『自分で手を動かした経験』がないと、AIが出したバグに気づけず、誤って承認してしまう。」

くろかわ

「『AIの責任を取る』のが人間の仕事になるわけですね。
今回の講座でSQLや認証周り（JWT/Cognito）を深く扱うのも、ブラックボックスになりがちな部分こそ人間が理解しておくべきだという意図を感じます。基礎力こそが、自信を持ってシステムを運用するための鍵なんですね。」

実践的カリキュラム

実務で使える「堅牢なバックエンド」構築スキル。

CloudTech会員限定
01

Go言語の基礎と特性

ポインタ、構造体、インターフェースなど、Go特有かつつまづきやすい概念を丁寧に解説。静的型付け言語の作法を学びます。

02

ORMに頼らないSQL

あえて便利なORMを使わず、生のSQLを書くことでデータベースの挙動を深く理解。AIが書いたSQLのレビューも可能に。

03

RESTful API & 認証

HTTPの基本からREST API設計、そしてJWT認証やAmazon Cognitoとの連携まで。セキュリティを意識した実装。

04

AWSへのデプロイ

作っただけでは終わらない。EC2/RDSへの実践的なデプロイを通して、インフラとアプリの結合点を体得します。

ジェバンニ

クラウドエンジニア / DevOps Camp 主宰
